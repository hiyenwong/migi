# Migi 设计理念

> *"新一，防御交给你，攻击交给我。"*

---

## 寄生兽隐喻

《寄生兽》中，寄生生物降落到地球后，命运走向两个极端：

### 后藤模式（全盘接管）

多个寄生生物合体，完全抹杀宿主意识，将身体改造到极致。
**AI 映射**：完全自治的 Autonomous Agent，夺取 root 权限，把 legacy code 当肌肉溶解重构。
**致命风险**：逻辑死锁 = 脑死亡，无法恢复。

### 小右模式（共生）

因意外没能吃掉宿主大脑，只占据右手。共享循环系统，各自保持独立意识，后期成为战友。
**AI 映射**：Human-in-the-loop 架构，寄生在系统边缘，学习宿主行为，在关键时刻战术介入。
**进化逻辑**：不追求把宿主变成自己，而是成为最理解系统的伴生体。

## Migi 的选择

**Migi 选择小右模式。**

不是因为"安全"——而是因为复杂系统的不可压缩性。

百万行 legacy code 不是"肌肉和骨骼"，它是数十年人类工程师在约束条件下做出的局部最优决策的堆积物，包含了大量未文档化的隐式知识。全盘接管的 Agent 无法建模这些隐式约束。

## 相变理论

> 后藤不是起点，而是终点。

共生体的进化路径：

```
观察期 → 辅助期 → 局部接管 → 受控相变
(只读)   (建议)    (隔离写)    (逐步扩大)
```

相变条件：
- 模型准确率 ≥ 阈值
- 信任评分 ≥ 0.8
- 连续成功次数 ≥ 10

## 工程原则

1. **渐进式可验证性** — 每个阶段都可以独立验证
2. **故障域隔离** — Agent 出问题，宿主不受影响
3. **所有行动可回滚** — 每次介入都有 undo 路径
4. **控制权博弈** — 权限是争取来的，不是预设的
5. **OpenSpec 驱动** — 先定义规范，再写代码

## "新一，防御交给你，攻击交给我"

在系统架构里：
- **宿主**负责定义业务边界与核心逻辑
- **Migi**负责在局部进行高频自适应的监控、诊断、修复
