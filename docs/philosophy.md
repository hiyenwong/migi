# Migi 设计理念

> *"新一，防御交给你，攻击交给我。"*

---

## 寄生兽隐喻

《寄生兽》中，寄生生物降落到地球后，命运走向两个极端：

### 后藤模式（全盘接管）

多个寄生生物合体，完全抹杀宿主意识，将身体改造到极致。
**AI 映射**：完全自治的 Autonomous Agent，夺取 root 权限，把 legacy code 当肌肉溶解重构。
**致命风险**：逻辑死锁 = 脑死亡，无法恢复。

### 小右模式（共生）

因意外没能吃掉宿主大脑，只占据右手。共享循环系统，各自保持独立意识，后期成为战友。
**AI 映射**：Human-in-the-loop 架构，寄生在系统边缘，学习宿主行为，在关键时刻战术介入。
**进化逻辑**：不追求把宿主变成自己，而是成为最理解系统的伴生体。

## Migi 的选择

**Migi 选择小右模式。**

不是因为"安全"——而是因为复杂系统的不可压缩性。

百万行 legacy code 不是"肌肉和骨骼"，它是数十年人类工程师在约束条件下做出的局部最优决策的堆积物，包含了大量未文档化的隐式知识。全盘接管的 Agent 无法建模这些隐式约束。

## 相变理论

> 后藤不是起点，而是终点。

共生体的进化路径：

```
观察期 → 辅助期 → 局部接管 → 受控相变
(只读)   (建议)    (隔离写)    (逐步扩大)
```

相变条件：
- 模型准确率 ≥ 阈值
- 信任评分 ≥ 0.8
- 连续成功次数 ≥ 10

## 寄生之后：四阶段终极目标

越过 **ControlledTransition** 之后，Migi 的进化不会停止。

```
                    ┌─────────────────┐
                    │  受控相变        │
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ 自适应学习       │ ← LLM 语义分析 + 策略自优化
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ 宿主教育         │ ← 从直接操作过渡为教导
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ 优雅休眠         │ ← 宿主不再需要 Migi
                    └─────────────────┘
```

### 阶段二：自适应学习（Auto-adaptive Learning）

越过 ControlledTransition 后，Migi 不再依赖纯统计模型：
- **LLM 语义分析** — 用 LLM 做语义级异常分析，理解事件背后的业务含义
- **策略自优化** — 从历史结果学习哪些干预有效，自动调整策略权重
- **参数自适应** — 信任阈值、窗口大小等根据宿主行为动态调整

### 阶段三：宿主教育（Host Education）

小右不会永远替新一战斗——它会教会宿主自我修复。
- 从直接操作（Hotfix）过渡为生成预防性建议
- 从"我来做"变成"看你学会了吗"
- 目标是让宿主系统在没有 Migi 的情况下也能健康运行

### 阶段四：优雅休眠（Graceful Dormancy）

寄生者的最高境界是让宿主不再需要它。

当 Migi 验证了宿主完全掌握自我修复能力后：
1. 冻结信任状态（snapshot）
2. 保留只读观察模式
3. 自动降级到 Observation 阶段
4. 进入"休眠"——只在检测到新异常模式时唤醒

> **"宿主不需要我，就是我最成功的时刻。"**

## 工程原则

1. **渐进式可验证性** — 每个阶段都可以独立验证
2. **故障域隔离** — Agent 出问题，宿主不受影响
3. **所有行动可回滚** — 每次介入都有 undo 路径
4. **控制权博弈** — 权限是争取来的，不是预设的
5. **OpenSpec 驱动** — 先定义规范，再写代码

## "新一，防御交给你，攻击交给我"

在系统架构里：
- **宿主**负责定义业务边界与核心逻辑
- **Migi**负责在局部进行高频自适应的监控、诊断、修复
